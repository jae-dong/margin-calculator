올데이픽 AI 소싱계산기 WEB v5.0.8

이번 수정
- 스니커즈 계산 결과의 수동 기록 저장 버튼 완전 제거
- 사진 분석 성공 시 자동 기록 저장 방식 유지
- 기록에 저장된 상품 사진을 누르면 검은 배경 팝업으로 크게 표시
- 팝업의 닫기 버튼 또는 사진 바깥을 눌러 닫기
- 스니커즈 기록의 사진 클릭은 확대 보기, 상품 정보 영역 클릭은 KREAM 검색으로 분리
- 가격표에 적힌 할인된 가격 입력 방식은 그대로 유지
- 추가 할인율을 0%부터 50%까지 5% 단위로 선택

적용
1. 압축을 풉니다.
2. GitHub 저장소 폴더의 기존 파일을 전부 교체합니다.
3. GitHub Desktop Summary에 WEB v5.0.8 추가할인 5단위·사진확대 입력 후 Commit, Push합니다.
4. Render 배포 완료 후 휴대폰 웹앱을 완전히 종료하고 다시 실행합니다.
5. 상단에 WEB v5.0.8이 표시되는지 확인합니다.

올데이픽 AI 소싱계산기 WEB v5.0.7

이번 수정
- KREAM 캡처 여러 장을 한 번에 서버로 보내지 않고, 각 사진을 가볍게 병렬 분석한 뒤 자동 통합합니다.
- 사진 해상도와 전송 용량을 낮춰 2장 분석 속도를 개선했습니다.
- 일부 사진만 성공해도 성공한 결과로 분석을 완료합니다.
- 모든 사진이 실패할 때만 2장 통합 분석을 마지막으로 1회 재시도합니다.
- 분석중 화면에 처리 완료 장수와 성공 장수를 표시합니다.
- Render의 30초 작업 종료 문제를 줄이도록 Gunicorn 스레드와 제한시간을 조정했습니다.

적용
1. 압축을 풉니다.
2. GitHub 저장소 폴더의 기존 파일을 전부 교체합니다.
3. GitHub Desktop에서 Summary에 WEB v5.0.7 KREAM 캡처 속도 안정화 입력 후 Commit, Push합니다.
4. Render 배포 완료를 확인합니다.
5. 휴대폰 웹앱을 완전히 종료한 후 다시 열고 WEB v5.0.7 표시를 확인합니다.


WEB v5.0.7 추가사항
- 일반상품/스니커즈 상품 사진 분석 성공 즉시 기록에 자동 저장
- KREAM 캡처 분석 성공 시 기존 동일 상품 자동기록을 최신 분석값으로 업데이트
- 기록 탭의 스니커즈 목록을 누르면 해당 상품코드로 KREAM 검색 연결
- 삭제 버튼은 KREAM 연결 없이 삭제만 실행


[v5.0.7 추가]
- 촬영/사진분석으로 자동 저장된 일반상품·스니커즈 기록에 대표 사진 썸네일 표시
- 사진은 휴대폰 로컬 저장공간 절약을 위해 자동 축소 저장
- KREAM 캡처 분석으로 기록을 업데이트해도 기존 상품 사진 유지
